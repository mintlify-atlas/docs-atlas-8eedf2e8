The Mastercad data model is defined in two layers: SQLAlchemy models in `database/schemas/datamodels.py` for the PostgreSQL schema, and Pydantic models in `api/schemas/models.py` for API request and response validation. This page documents the entities you will encounter in API responses.

## Company

The central entity. Represents a single fintech company with firmographic data, classification flags, and processing timestamps.

<ResponseField name="id" type="integer" required>
  Internal auto-incremented primary key.
</ResponseField>

<ResponseField name="forager_id" type="integer">
  External identifier from the Forager API. Unique when present.
</ResponseField>

<ResponseField name="name" type="string" required>
  Company name. Indexed for fast search.
</ResponseField>

<ResponseField name="website" type="string">
  Full website URL (up to 512 characters).
</ResponseField>

<ResponseField name="domain" type="string">
  Normalized domain (e.g., `stripe.com`). Unique index — used for deduplication and on-demand lookups.
</ResponseField>

<ResponseField name="logo" type="string">
  URL to the company logo image.
</ResponseField>

<ResponseField name="short_description" type="string">
  One-sentence value proposition generated by the Classifier Agent.
</ResponseField>

<ResponseField name="description" type="string">
  Full company description. May be translated to English by DeepL before enrichment.
</ResponseField>

<ResponseField name="operating_status" type="string">
  Current operating status (e.g., `active`, `closed`).
</ResponseField>

<ResponseField name="founded_date" type="date">
  Company founding date.
</ResponseField>

<ResponseField name="employees_range" type="string">
  Headcount range string (e.g., `51-200`).
</ResponseField>

<ResponseField name="employees_amount" type="integer">
  Exact employee count when available.
</ResponseField>

<ResponseField name="location_text" type="string">
  Free-text location (e.g., `San Francisco, CA`).
</ResponseField>

<ResponseField name="location_continent" type="string">
  Continent derived from the company's primary location.
</ResponseField>

<ResponseField name="revenue" type="decimal">
  Annual revenue in USD (18 digits, 2 decimal places).
</ResponseField>

<ResponseField name="is_fintech" type="boolean">
  Set by the Classifier Agent. `true` if the company is a fintech, `false` if definitively not, `null` if unknown.
</ResponseField>

<ResponseField name="is_bank" type="boolean">
  Set by the Bank Classifier. `true` if the company is primarily a bank rather than a fintech startup.
</ResponseField>

<ResponseField name="linkedin_profile_url" type="string">
  LinkedIn company page URL.
</ResponseField>

<ResponseField name="linkedin_industry" type="string">
  Industry label from LinkedIn.
</ResponseField>

<ResponseField name="revenue_models" type="string">
  Comma-separated revenue model tags (e.g., `SaaS, Transaction fee`).
</ResponseField>

<ResponseField name="customer_category" type="string">
  Primary customer type (e.g., `B2B`, `B2C`, `B2B2C`).
</ResponseField>

<ResponseField name="keywords" type="Keyword[]">
  <Expandable title="Keyword object">
    <ResponseField name="id" type="integer" required>Internal keyword ID.</ResponseField>
    <ResponseField name="forager_id" type="integer">Forager external ID.</ResponseField>
    <ResponseField name="name" type="string" required>Keyword label (e.g., `fintech`, `payments`).</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="funding_rounds" type="FundingRound[]">
  Ordered by `featured_date` descending. See [FundingRound](#fundinground) below.
</ResponseField>

<ResponseField name="subcategories" type="SubcategoryInfo[]">
  Fintech subcategories assigned by the Classifier Agent. See [SubcategoryInfo](#subcategoryinfo) below.
</ResponseField>

<ResponseField name="origin_source_id" type="integer">
  Reference to the data source that first created this record (`1` = Forager, `2` = VC Scraper, `3` = Blumb, `4` = Mastercard).
</ResponseField>

<ResponseField name="forager_updated_at" type="datetime">
  Timestamp of the last update received from Forager.
</ResponseField>

<ResponseField name="created_at" type="datetime" required>
  Record creation timestamp (UTC).
</ResponseField>

<ResponseField name="updated_at" type="datetime" required>
  Last modification timestamp (UTC).
</ResponseField>

## CompanyDetail

Extends `Company` with calculated investment metrics, investor relationships, and office locations. Returned by the single-company detail endpoint.

<ResponseField name="investment_raised" type="decimal">
  Total capital raised across all funding rounds (sum of `FundingRound.amount`).
</ResponseField>

<ResponseField name="investment_stage" type="string">
  Current funding stage derived from the most recent round (e.g., `Series B`).
</ResponseField>

<ResponseField name="last_round" type="FundingRound">
  The most recent funding round. See [FundingRound](#fundinground) below.
</ResponseField>

<ResponseField name="origin_source" type="Source">
  <Expandable title="Source object">
    <ResponseField name="id" type="integer" required>Source ID.</ResponseField>
    <ResponseField name="name" type="string" required>Source name (e.g., `forager`).</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="investors" type="Investor[]">
  <Expandable title="Investor object">
    <ResponseField name="id" type="integer" required>Internal investor ID.</ResponseField>
    <ResponseField name="name" type="string" required>Investor name.</ResponseField>
    <ResponseField name="website" type="string">Investor website URL.</ResponseField>
    <ResponseField name="domain" type="string">Normalized investor domain.</ResponseField>
    <ResponseField name="global_hq" type="string">Investor headquarters location.</ResponseField>
    <ResponseField name="countries_investment" type="string">Comma-separated countries where the investor is active.</ResponseField>
    <ResponseField name="description" type="string">Investor description.</ResponseField>
    <ResponseField name="type" type="string">Investor type (e.g., `VC`, `PE`, `Angel`).</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="offices" type="CompanyOfficeInfo[]">
  All offices including HQ (`office_type = 'hq'`) and branches.
  <Expandable title="CompanyOfficeInfo object">
    <ResponseField name="id" type="integer" required>Office ID.</ResponseField>
    <ResponseField name="office_type" type="string" required>`hq` or `branch`.</ResponseField>
    <ResponseField name="city" type="string">City name.</ResponseField>
    <ResponseField name="state" type="string">State or region.</ResponseField>
    <ResponseField name="street_address" type="string">Street address.</ResponseField>
    <ResponseField name="postcode" type="string">Postal code.</ResponseField>
    <ResponseField name="country" type="CountryInfo">
      <Expandable title="CountryInfo object">
        <ResponseField name="id" type="integer" required>Country ID.</ResponseField>
        <ResponseField name="code" type="string" required>ISO 3166-1 alpha-2 code (e.g., `US`, `GB`).</ResponseField>
        <ResponseField name="name" type="string" required>Country name.</ResponseField>
        <ResponseField name="region" type="string">Region code: `AP`, `EMEA`, `LATAM`, or `NA`.</ResponseField>
        <ResponseField name="division" type="string">Sub-region (e.g., `Western Europe`, `South East Asia`).</ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

## FundingRound

Represents a single capital raise event linked to a company.

<ResponseField name="id" type="integer" required>
  Internal funding round ID.
</ResponseField>

<ResponseField name="forager_id" type="integer">
  Forager external ID for the round.
</ResponseField>

<ResponseField name="company_id" type="integer" required>
  Foreign key to the parent company.
</ResponseField>

<ResponseField name="source" type="string" required>
  Data source for this round (e.g., `forager`).
</ResponseField>

<ResponseField name="featured_date" type="date">
  Date the round was announced or closed.
</ResponseField>

<ResponseField name="funding_type" type="string" required>
  Round type. Possible values include: `seed`, `series_a`, `series_b`, `series_c`, `pre_seed`, `angel`, `convertible_note`, `private_equity`, `grant`, `debt_financing`, and others.
</ResponseField>

<ResponseField name="amount" type="decimal">
  Amount raised in USD (18 digits, 2 decimal places).
</ResponseField>

## SubcategoryInfo and CategoryInfo

Fintech taxonomy nodes assigned to companies by the Classifier Agent.

<ResponseField name="id" type="integer" required>
  Subcategory ID.
</ResponseField>

<ResponseField name="name" type="string" required>
  Subcategory name (e.g., `Card Payments`, `Digital Wallets`).
</ResponseField>

<ResponseField name="category_id" type="integer">
  Foreign key to the parent category.
</ResponseField>

<ResponseField name="category" type="CategoryInfo">
  <Expandable title="CategoryInfo object">
    <ResponseField name="id" type="integer" required>Category ID.</ResponseField>
    <ResponseField name="name" type="string" required>Category name (e.g., `Payments`, `Digital Banking`).</ResponseField>
  </Expandable>
</ResponseField>

## CompanyFilter

Used as the request body for `POST /v1/companies/filter`. All fields are optional — omitting a field means no filter is applied for that dimension.

<ResponseField name="search" type="string">
  Free-text search across company name, domain, subcategory name, and keyword name. Minimum 2 characters. Domain format is detected automatically.
</ResponseField>

<ResponseField name="categories" type="integer[]">
  Filter by one or more category IDs. Returns companies that have at least one subcategory belonging to any of the specified categories.
</ResponseField>

<ResponseField name="subcategories" type="integer[]">
  Filter by one or more subcategory IDs directly.
</ResponseField>

<ResponseField name="keyword_ids" type="integer[]">
  Filter by keyword IDs. Returns companies tagged with any of the specified keywords.
</ResponseField>

<ResponseField name="country_ids" type="integer[]">
  Filter by country IDs. Matches companies whose HQ or any office is located in the specified countries.
</ResponseField>

<ResponseField name="employees_range" type="string">
  Filter by headcount range string (e.g., `51-200`).
</ResponseField>

<ResponseField name="funding_types" type="string[]">
  Filter by funding round types. Returns companies that have at least one round of the specified type.
</ResponseField>

<ResponseField name="location_continent" type="string">
  Filter by continent (e.g., `North America`, `Europe`).
</ResponseField>

<ResponseField name="origin_source_id" type="integer[]">
  Filter by data source IDs.
</ResponseField>

<ResponseField name="page" type="integer">
  Page number for pagination. Starts at `1`. Default: `1`.
</ResponseField>

<ResponseField name="page_size" type="integer">
  Results per page. Between `1` and `1000`. Default: `50`.
</ResponseField>

## Fintech category taxonomy

The classifier uses a two-level taxonomy of 10 categories and 57 subcategories. The top-level categories are:

| Category | Description |
|----------|-------------|
| **Payments** | Card payments, P2P, payment gateways, installments, international transfers |
| **Digital Banking** | Banking-as-a-service, digital wallets, digital onboarding, branch digitalization |
| **Lending** | Consumer lending, business lending, credit scoring, underwriting |
| **Risk Management and Compliance** | Fraud detection, KYC/AML, cyber security, digital identity, regulatory compliance |
| **Big Data and Open Banking** | Open banking, data aggregation, APIs and interoperability, analytics |
| **Commercial and SMB** | Cash flow management, enterprise solutions, SMB tools, processing, revenue management |
| **Customer Engagement and Servicing** | Loyalty, personal finance management, wealth management, insurance, subscription billing |
| **Retail and E-Commerce** | Merchant solutions, mobile POS, bill pay, e-commerce, physical retail |
| **Emerging Technologies and Innovations** | Blockchain, crypto, AI/ML, NFTs, IoT, digital asset infrastructure |
| **Social Impact and Sustainability** | Financial inclusion, financial education, sustainability, smart cities |

<Tip>
  To browse the full list of subcategories with their descriptions, call `GET /v1/companies/filter` which returns all available categories and subcategories with their IDs.
</Tip>
