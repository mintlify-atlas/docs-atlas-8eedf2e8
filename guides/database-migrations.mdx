Mastercad Backend uses [Alembic](https://alembic.sqlalchemy.org/) to manage PostgreSQL schema
changes. Migration scripts live in `migrations/versions/` and are applied in order against the
target database.

## Directory structure

```
├── alembic.ini              # Alembic configuration file
└── migrations/
    ├── env.py               # Migration environment — loads DB URL from .env
    └── versions/            # Auto-generated migration scripts
        └── <rev>_<slug>.py
```

`alembic.ini` sets `script_location = migrations`. The `sqlalchemy.url` entry is left blank;
`migrations/env.py` reads `DB_URL` (or the individual `POSTGRES_*` variables) from the environment
at runtime.

## First-run setup

<Steps>
  <Step title="Ensure the database is running">
    If you are using Docker Compose, start the stack first:

    ```bash
    docker compose up --build
    ```

    The `app` service waits for the `db` health check to pass before accepting commands.
  </Step>

  <Step title="Apply all migrations">
    Run `alembic upgrade head` inside the container to create all tables defined in the migration
    history.

    ```bash
    docker compose exec app alembic upgrade head
    ```

    Running this command on a fresh database creates every table from scratch. On an existing
    database it applies only the migrations that have not been applied yet.
  </Step>

  <Step title="Verify the schema">
    Connect to the database and confirm the tables exist:

    ```bash
    docker compose exec db psql -U $POSTGRES_USER -d $POSTGRES_DB -c '\dt'
    ```
  </Step>
</Steps>

## Common commands

<CodeGroup>

```bash Apply all pending migrations
alembic upgrade head
```

```bash Create a new autogenerated migration
alembic revision --autogenerate -m "add unique constraints"
```

```bash Revert the last migration
alembic downgrade -1
```

```bash Show current revision
alembic current
```

```bash Show migration history
alembic history --verbose
```

</CodeGroup>

### Running inside Docker

Prefix any Alembic command with `docker compose exec app` to run it in the running container:

```bash
docker compose exec app alembic upgrade head
docker compose exec app alembic revision --autogenerate -m "add company index"
docker compose exec app alembic downgrade -1
```

## Creating a new migration

<Steps>
  <Step title="Edit the SQLAlchemy models">
    Make your schema changes in `database/schemas/datamodels.py`. Alembic's autogenerate feature
    compares the current database state against the SQLAlchemy metadata to produce the migration
    diff.
  </Step>

  <Step title="Generate the migration file">
    ```bash
    docker compose exec app alembic revision --autogenerate -m "describe your change"
    ```

    A new file is created under `migrations/versions/`. Review it before applying — autogenerate
    does not detect every type of change (e.g., column renames, constraint changes).
  </Step>

  <Step title="Apply the migration">
    ```bash
    docker compose exec app alembic upgrade head
    ```
  </Step>
</Steps>

---

## Reverting a migration

To roll back the most recently applied migration:

```bash
docker compose exec app alembic downgrade -1
```

To roll back to a specific revision:

```bash
docker compose exec app alembic downgrade <revision_id>
```

<Warning>
  Always back up your production database before running `alembic upgrade` or `alembic downgrade`.
  Downgrade scripts may be destructive — dropping columns or tables cannot be undone without a
  restore.
</Warning>
