Mastercad Backend is configured entirely through environment variables. Copy `env.example` to `.env`
in the repository root and fill in the values before starting the application.

```bash
cp env.example .env
```

<Warning>
  Several variables are validated at startup. If `SUPABASE_URL`, `SUPABASE_KEY`,
  `SUPABASE_JWT_SECRET`, `OPENAI_API_KEY`, or `PERPLEXITY_API_KEY` are missing or empty, the
  application will raise a `ValueError` and refuse to start.
</Warning>

---

## Database

The application constructs its PostgreSQL connection string from the five variables below. You can
also set `DB_URL` directly to override the constructed URL.

<ParamField path="POSTGRES_USER" type="string" required>
  PostgreSQL username. Default: `usuario`.
</ParamField>

<ParamField path="POSTGRES_PASSWORD" type="string" required>
  PostgreSQL password. Default: `password`.
</ParamField>

<ParamField path="POSTGRES_DB" type="string" required>
  Database name. Default: `mastercad_db`.
</ParamField>

<ParamField path="POSTGRES_HOST" type="string">
  Hostname or IP of the Postgres server. Default: `db` (matches the Docker Compose service name).
</ParamField>

<ParamField path="POSTGRES_PORT" type="string">
  Postgres port. Default: `5432`.
</ParamField>

<ParamField path="DB_URL" type="string">
  Full SQLAlchemy connection URL. When set, this value takes precedence over the individual
  `POSTGRES_*` variables. Format: `postgresql+psycopg2://user:password@host:port/dbname`.
</ParamField>

## Authentication

Mastercad Backend delegates authentication to Supabase. All three variables are required.

<ParamField path="SUPABASE_URL" type="string" required>
  URL of your Supabase project (e.g., `https://xyzcompany.supabase.co`).
</ParamField>

<ParamField path="SUPABASE_KEY" type="string" required>
  Supabase service role or anon key used for server-side calls.
</ParamField>

<ParamField path="SUPABASE_JWT_SECRET" type="string" required>
  JWT secret used to verify tokens issued by Supabase Auth.
</ParamField>

---

## AI services

<ParamField path="OPENAI_API_KEY" type="string" required>
  API key for OpenAI. Used for enrichment and classification agents. The application uses `gpt-4o`
  by default.
</ParamField>

<ParamField path="PERPLEXITY_API_KEY" type="string" required>
  API key for Perplexity AI. Used for `sonar-pro` and `sonar-deep-research` model calls.
</ParamField>

<ParamField path="DEEPL_API_KEY" type="string">
  API key for DeepL. Used for translating company descriptions. Optional â€” translation steps are
  skipped if this key is absent.
</ParamField>

## Data sources

<ParamField path="FORAGER_API_KEY" type="string">
  API key for Forager. Required for the `discovery` ETL command.
</ParamField>

<ParamField path="FORAGER_ACCOUNT_ID" type="string">
  Account ID for the Forager API.
</ParamField>

<ParamField path="FORAGER_BASE_URL" type="string">
  Base URL for the Forager API. Default: `https://api-v2.forager.ai/api`.
</ParamField>

<ParamField path="TAVILY_API_KEY" type="string">
  API key for Tavily web search. Used in research and enrichment agents.
</ParamField>

<ParamField path="URL_COLLECTOR" type="string">
  Base URL for the external URL collector service used during data ingestion.
</ParamField>

---

## Storage

Logos and generated PPTX reports are stored in separate S3 buckets.

<ParamField path="S3_ACCESS_KEY_ID" type="string">
  AWS access key ID for S3 operations.
</ParamField>

<ParamField path="S3_SECRET_ACCESS_KEY" type="string">
  AWS secret access key for S3 operations.
</ParamField>

<ParamField path="S3_BUCKET_LOGOS" type="string">
  S3 bucket name for company logos. Default: `market-scan-logos`.
</ParamField>

<ParamField path="S3_BUCKET_REPORTS" type="string">
  S3 bucket name for PPTX report files. Default: `market-scan-reportes`.
</ParamField>

<ParamField path="AWS_REGION" type="string">
  AWS region for S3 and other AWS services. Default: `eu-west-1`.
</ParamField>

---

## Observability

Langfuse tracing is enabled automatically when both keys are present. If either key is missing,
tracing is silently disabled.

<ParamField path="LANGFUSE_PUBLIC_KEY" type="string">
  Langfuse public key for LLM tracing.
</ParamField>

<ParamField path="LANGFUSE_SECRET_KEY" type="string">
  Langfuse secret key for LLM tracing.
</ParamField>

<ParamField path="LANGFUSE_HOST" type="string">
  Langfuse host URL. Default: `https://cloud.langfuse.com`.
</ParamField>

---

## Integrations

<ParamField path="SLACK_WEBHOOK_URL" type="string">
  Incoming webhook URL for Slack notifications. Optional.
</ParamField>

<ParamField path="GAMMA_API_KEY" type="string">
  API key for the Gamma presentation generation service.
</ParamField>

<ParamField path="GAMMA_TEMPLATE_ID" type="string">
  Template ID used when generating presentations via the Gamma API.
</ParamField>

---

## Runtime

<ParamField path="ENVIRONMENT" type="string">
  Deployment environment. Accepted values: `development`, `staging`, `production`. Default:
  `development`.
</ParamField>

<ParamField path="LOG_LEVEL" type="string">
  Application log level. Accepted values: `DEBUG`, `INFO`, `WARNING`, `ERROR`. Default: `DEBUG`.
</ParamField>

---

## Example `.env`

```bash
# Database
POSTGRES_USER=mastercad
POSTGRES_PASSWORD=changeme
POSTGRES_DB=mastercad_db
POSTGRES_HOST=db
POSTGRES_PORT=5432

# Authentication (required)
SUPABASE_URL=https://xyzcompany.supabase.co
SUPABASE_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
SUPABASE_JWT_SECRET=super-secret-jwt-secret

# AI services (required)
OPENAI_API_KEY=sk-...
PERPLEXITY_API_KEY=pplx-...
DEEPL_API_KEY=

# Data sources
FORAGER_API_KEY=
FORAGER_ACCOUNT_ID=
FORAGER_BASE_URL=https://api-v2.forager.ai/api
TAVILY_API_KEY=

# Storage
S3_ACCESS_KEY_ID=
S3_SECRET_ACCESS_KEY=
S3_BUCKET_LOGOS=market-scan-logos
S3_BUCKET_REPORTS=market-scan-reportes
AWS_REGION=eu-west-1

# Observability
LANGFUSE_PUBLIC_KEY=
LANGFUSE_SECRET_KEY=
LANGFUSE_HOST=https://cloud.langfuse.com

# Integrations
SLACK_WEBHOOK_URL=
GAMMA_API_KEY=
GAMMA_TEMPLATE_ID=

# Runtime
ENVIRONMENT=development
LOG_LEVEL=DEBUG
```
