---
title: "Market Research"
description: "Generate AI-powered market research reports, export to PPTX or Gamma, and extract VC portfolio data."
---

All endpoints in this group require a valid Bearer token.

```
Authorization: Bearer <your_access_token>
```

<Warning>
  You can generate a maximum of **200 reports per month**. Requests that exceed this limit return HTTP `429 Too Many Requests`. The current usage count is included in the `GET /v1/market-research/reports` response under `monthly_reports_used`.
</Warning>

---

## POST /v1/market-research

Generate a full AI-written market research report for 1–5 specific companies within a given topic. The API fetches full company profiles from the database, runs them through the market report agent, saves the report, and returns the result synchronously.

**Authentication:** Bearer token required

<Warning>
  This endpoint counts against your 200 monthly report limit.
</Warning>

### Request body

<ParamField body="topic" type="string" required>
  The market topic or industry context for the report (e.g. `"European B2B payments infrastructure"`).
</ParamField>

<ParamField body="company_ids" type="integer[]" required>
  List of 1–5 internal company IDs to include in the report. Retrieve IDs from `POST /v1/companies/filter`.
</ParamField>

### Response fields

<ResponseField name="success" type="boolean" required>
  `true` when the report was generated successfully.
</ResponseField>

<ResponseField name="data" type="object" required>
  Report data and metadata.

  <Expandable title="data">
    <ResponseField name="report" type="object" required>
      Structured market research report object containing sections, analysis, and findings.
    </ResponseField>
    <ResponseField name="companies" type="object[]" required>
      Array of company detail objects included in the report.
    </ResponseField>
    <ResponseField name="sources" type="string[]">
      List of source URLs cited in the report.
    </ResponseField>
    <ResponseField name="metadata" type="object" required>
      Report metadata.
      <Expandable title="metadata">
        <ResponseField name="report_id" type="integer">Database ID of the saved report.</ResponseField>
        <ResponseField name="topic" type="string">Topic used to generate the report.</ResponseField>
        <ResponseField name="companies_provided" type="integer">Number of companies included.</ResponseField>
        <ResponseField name="timestamp" type="string">ISO 8601 generation timestamp.</ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

### Example

<CodeGroup>

```bash Request
curl -X POST http://localhost:8000/v1/market-research \
  -H "Authorization: Bearer <your_access_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "topic": "European B2B payments infrastructure",
    "company_ids": [42, 88, 137]
  }'
```

```json Response
{
  "success": true,
  "data": {
    "report": {
      "executive_summary": "...",
      "market_overview": "...",
      "company_profiles": []
    },
    "companies": [],
    "sources": [
      "https://example.com/source1",
      "https://example.com/source2"
    ],
    "metadata": {
      "report_id": 201,
      "topic": "European B2B payments infrastructure",
      "companies_provided": 3,
      "timestamp": "2026-04-08T14:00:00.000000"
    }
  },
  "errors": null
}
```

</CodeGroup>

## POST /v1/market-research/reports/{report_id}/start-gamma

Start an asynchronous Gamma presentation generation job and return immediately with a `generationId`. The client must poll `GET /v1/market-research/reports/{report_id}/gamma-status/{generation_id}` until the status transitions to `completed` or `failed`.

If a Gamma presentation has already been generated for this report, the endpoint returns `status: "completed"` and the `exportUrl` directly without re-triggering generation.

**Authentication:** Bearer token required

<Note>
  This endpoint uses an async poll pattern. After calling `start-gamma`, use the returned `generationId` to poll the `gamma-status` endpoint every few seconds until `status` is `"completed"` or `"failed"`.
</Note>

### Path parameters

<ParamField path="report_id" type="integer" required>
  The report ID to start Gamma generation for.
</ParamField>

### Response fields

<ResponseField name="success" type="boolean" required>
  `true` when the job was started (or already completed).
</ResponseField>

<ResponseField name="data" type="object" required>
  Generation job result.

  <Expandable title="data — pending">
    <ResponseField name="status" type="string" required>
      `"pending"` when the job has been queued. Poll the status endpoint.
    </ResponseField>
    <ResponseField name="generationId" type="string" required>
      Opaque generation job ID. Pass this to `GET .../gamma-status/{generation_id}`.
    </ResponseField>
  </Expandable>

  <Expandable title="data — already generated">
    <ResponseField name="status" type="string" required>
      `"completed"` when a Gamma file was previously generated.
    </ResponseField>
    <ResponseField name="exportUrl" type="string" required>
      Direct Gamma export URL.
    </ResponseField>
  </Expandable>
</ResponseField>

### Example

<CodeGroup>

```bash Request
curl -X POST http://localhost:8000/v1/market-research/reports/201/start-gamma \
  -H "Authorization: Bearer <your_access_token>"
```

```json Response — job started
{
  "success": true,
  "data": {
    "status": "pending",
    "generationId": "gen_a1b2c3d4e5f6"
  },
  "errors": null
}
```

```json Response — already generated
{
  "success": true,
  "data": {
    "status": "completed",
    "exportUrl": "https://gamma.app/export/abc123.pptx"
  },
  "errors": null
}
```

</CodeGroup>

## GET /v1/market-research/reports/{report_id}/gamma-status/{generation_id}

Poll the status of a Gamma generation job started by `POST .../start-gamma`. When status transitions to `completed`, the `exportUrl` is returned and saved to the report.

**Authentication:** Bearer token required

### Path parameters

<ParamField path="report_id" type="integer" required>
  The report ID the generation job belongs to.
</ParamField>

<ParamField path="generation_id" type="string" required>
  The `generationId` returned by `POST .../start-gamma`.
</ParamField>

### Response fields

<ResponseField name="success" type="boolean" required>
  `true` when the status was retrieved.
</ResponseField>

<ResponseField name="data" type="object" required>
  Current generation status.

  <Expandable title="data">
    <ResponseField name="status" type="string" required>
      Current status: `"pending"`, `"completed"`, or `"failed"`.
    </ResponseField>
    <ResponseField name="exportUrl" type="string">
      Direct Gamma download URL. Present only when `status` is `"completed"`.
    </ResponseField>
  </Expandable>
</ResponseField>

### Example

<CodeGroup>

```bash Request
curl "http://localhost:8000/v1/market-research/reports/201/gamma-status/gen_a1b2c3d4e5f6" \
  -H "Authorization: Bearer <your_access_token>"
```

```json Response — in progress
{
  "success": true,
  "data": {
    "status": "pending"
  },
  "errors": null
}
```

```json Response — complete
{
  "success": true,
  "data": {
    "status": "completed",
    "exportUrl": "https://gamma.app/export/abc123.pptx"
  },
  "errors": null
}
```

</CodeGroup>

    <ResponseField name="sources" type="string[]">
      Source URLs cited in the report.
