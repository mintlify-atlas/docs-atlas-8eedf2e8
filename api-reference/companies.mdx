## GET /v1/companies

Return all companies in the database without any filtering. Useful for small datasets or initial data loads. For production use, prefer `POST /v1/companies/filter` to paginate results.

**Authentication:** Bearer token required

### Response fields

<ResponseField name="success" type="boolean" required>
  `true` when the request succeeded.
</ResponseField>

<ResponseField name="data" type="object" required>
  Paginated list of companies.

  <Expandable title="data">
    <ResponseField name="companies" type="Company[]" required>
      Array of company objects. See the [Company object](#company-object) below.
    </ResponseField>
    <ResponseField name="total" type="integer" required>
      Total number of companies matching the query.
    </ResponseField>
    <ResponseField name="page" type="integer" required>
      Current page number.
    </ResponseField>
    <ResponseField name="page_size" type="integer" required>
      Number of results per page.
    </ResponseField>
    <ResponseField name="total_pages" type="integer" required>
      Total number of available pages.
    </ResponseField>
  </Expandable>
</ResponseField>

### Example

<CodeGroup>

```bash Request
curl http://localhost:8000/v1/companies \
  -H "Authorization: Bearer <your_access_token>"
```

```json Response
{
  "success": true,
```

## POST /v1/companies/filter

Filter and paginate companies by one or more criteria. Supports text search, category filtering, geography, employee count, funding type, and keyword matching.

**Authentication:** Bearer token required

### Request body

<ParamField body="search" type="string">
  Free-text search across company name, domain, subcategory, and keywords. Minimum 2 characters. Automatically detects domain format (e.g. `stripe.com`).
</ParamField>

<ParamField body="categories" type="integer[]">
  List of category IDs to filter by. Retrieve available IDs from `GET /v1/companies/filter`.
</ParamField>

<ParamField body="subcategories" type="integer[]">
  List of subcategory IDs to filter by.
</ParamField>

<ParamField body="keyword_ids" type="integer[]">
  List of keyword IDs to filter by. Search for keywords by name using `GET /v1/companies/keywords/search`.
</ParamField>

<ParamField body="country_ids" type="integer[]">
  List of country IDs to filter by. Matches companies with a headquarters or office in any of the specified countries.
</ParamField>

<ParamField body="employees_range" type="string">
  Employee headcount band (e.g. `"50 - 200"`). Valid values returned by `GET /v1/companies/filter`.
</ParamField>

<ParamField body="funding_types" type="string[]">
  List of funding round types to include (e.g. `["Series A", "Series B"]`).
</ParamField>

<ParamField body="location_continent" type="string">
  Filter by continent (e.g. `"Europe"`, `"North America"`).
</ParamField>

<ParamField body="origin_source_id" type="integer[]">
  List of data source IDs to filter by (1 = Forager, 2 = VC Scraper, 3 = Blumb, 4 = Mastercard).
</ParamField>

<ParamField body="page" type="integer" default="1">
  Page number to retrieve. Starts at 1.
</ParamField>

<ParamField body="page_size" type="integer" default="50">
  Number of results per page. Maximum 1000.
</ParamField>

### Response fields

<ResponseField name="success" type="boolean" required>
  `true` when the request succeeded.
</ResponseField>

<ResponseField name="data" type="object" required>
  Filtered and paginated company list.

  <Expandable title="data">
    <ResponseField name="companies" type="Company[]" required>
      Array of matching company objects. See the [Company object](#company-object) below.
    </ResponseField>
    <ResponseField name="total" type="integer" required>
      Total number of companies matching the applied filters.
    </ResponseField>
    <ResponseField name="page" type="integer" required>
      Current page number.
    </ResponseField>
    <ResponseField name="page_size" type="integer" required>
      Page size used for this response.
    </ResponseField>
    <ResponseField name="total_pages" type="integer" required>
      Total number of pages available for the current filter set.
    </ResponseField>
  </Expandable>
</ResponseField>

### Example

<CodeGroup>

```bash Request
curl -X POST http://localhost:8000/v1/companies/filter \
  -H "Authorization: Bearer <your_access_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "search": "payments",
    "categories": [3],
    "location_continent": "Europe",
    "employees_range": "50 - 200",
    "page": 1,
    "page_size": 20
  }'
```

```json Response
{
  "success": true,
  "data": {
    "companies": [
      {
        "id": 42,
        "name": "Mollie",
        "website": "https://www.mollie.com",
        "domain": "mollie.com",
        "description": "Payment service provider for European businesses.",
        "employees_range": "50 - 200",
        "location_continent": "Europe",
        "is_fintech": true,
        "created_at": "2024-02-10T09:00:00Z",
        "updated_at": "2026-03-15T11:00:00Z"
      }
    ],
    "total": 87,
    "page": 1,
    "page_size": 20,
    "total_pages": 5
  },
  "errors": null
}
```

</CodeGroup>

## POST /v1/companies/enrich

Trigger firmographic enrichment for one or more companies. Fetches updated data for the specified fields from external sources and writes the results back to the database.

**Authentication:** Bearer token required

<Info>
  Provide either `company_ids` (a list of specific IDs) or `id_range` (an inclusive start–end range). At least one is required.
</Info>

### Request body

<ParamField body="company_ids" type="integer[]">
  List of internal company IDs to enrich. Can be combined with `id_range`.
</ParamField>

<ParamField body="id_range" type="object">
  Inclusive range of company IDs to enrich.

  <Expandable title="id_range">
    <ParamField body="start" type="integer" required>
      First company ID in the range (≥ 1).
    </ParamField>
    <ParamField body="end" type="integer" required>
      Last company ID in the range (≥ start).
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="fields" type="string[]">
  Subset of firmographic fields to update. When omitted or empty, all available fields are updated. Example: `["employees_range", "revenue", "linkedin_industry"]`.
</ParamField>

### Response fields

<ResponseField name="success" type="boolean" required>
  `true` when the enrichment job was accepted and completed.
</ResponseField>

<ResponseField name="data" type="object" required>
  Enrichment result summary returned by the enrichment agent.
</ResponseField>

### Example

<CodeGroup>

```bash Enrich specific IDs
curl -X POST http://localhost:8000/v1/companies/enrich \
  -H "Authorization: Bearer <your_access_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "company_ids": [42, 43, 44],
    "fields": ["employees_range", "revenue"]
  }'
```

```bash Enrich by ID range
curl -X POST http://localhost:8000/v1/companies/enrich \
  -H "Authorization: Bearer <your_access_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "id_range": { "start": 100, "end": 200 }
  }'
```

```json Response
{
  "success": true,
  "data": {
    "enriched": 3,
    "failed": 0,
    "skipped": 0
  },
  "errors": null
}
```

</CodeGroup>

## POST /v1/companies/add

Search for a company in Forager by website (and optionally name), save it to the database, then queue background processing (translation, location extraction, and classification). The endpoint responds immediately — background tasks run asynchronously.

**Authentication:** Bearer token required

### Request body

<ParamField body="website" type="string" required>
  Company website URL. Will be normalized to a domain before lookup (e.g. `https://stripe.com` → `stripe.com`).
</ParamField>

<ParamField body="name" type="string">
  Optional company name to improve search accuracy. Minimum 2 characters, maximum 255.
</ParamField>

### Response fields

<ResponseField name="success" type="boolean" required>
  `true` regardless of whether the company was found.
</ResponseField>

<ResponseField name="data" type="object" required>
  Search result.

  <Expandable title="data">
    <ResponseField name="found" type="boolean" required>
      `true` if the company was found in Forager and saved to the database.
    </ResponseField>
    <ResponseField name="company" type="Company">
      The saved company object when `found` is `true`. `null` when not found. See [Company object](#company-object).
    </ResponseField>
    <ResponseField name="message" type="string">
      Human-readable status message (e.g. `"Company added (ID: 42). Processing in background."`).
    </ResponseField>
  </Expandable>
</ResponseField>

### Example

<CodeGroup>

```bash Request
curl -X POST http://localhost:8000/v1/companies/add \
  -H "Authorization: Bearer <your_access_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Mollie",
    "website": "https://www.mollie.com"
  }'
```

```json Company found
{
  "success": true,
  "data": {
    "found": true,
    "company": {
      "id": 42,
      "name": "Mollie",
      "website": "https://www.mollie.com",
      "domain": "mollie.com",
      "created_at": "2026-04-08T10:00:00Z",
      "updated_at": "2026-04-08T10:00:00Z"
    },
    "message": "Company added (ID: 42). Processing in background."
  },
  "errors": null
}
```

```json Company not found
{
  "success": true,
  "data": {
    "found": false,
    "company": null,
    "message": "Company not found"
  },
  "errors": null
}
```

</CodeGroup>

<Note>
  When `found` is `true`, background processing (translation, location extraction, category classification) is queued and runs after the response is returned. The company record will be complete within a few seconds.
</Note>

    <ResponseField name="companies_analyzed" type="integer" required>

</ResponseField>
